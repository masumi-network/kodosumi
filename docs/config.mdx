This document describes all available configuration settings for Kodosumi. Settings can be configured through environment variables, a `.env` file, or command-line arguments.

## Environment Variables

All configuration settings can be set using environment variables with the prefix `KODO_`. For example, to set the execution directory, use `KODO_EXEC_DIR`. These environment variables can be overridden using the `koco` command-line tool.

## Configuration Settings

### Execution Settings

- `EXEC_DIR` (default: `"./data/execution"`)
  - Directory where execution files are stored
  - Will be created automatically if it doesn't exist
  - `koco` option `--exec-dir` (available in all commands)
    - Can override `KODO_EXEC_DIR` in all commands

### Spooler Settings

- `SPOOLER_LOG_FILE` (default: `"./data/spooler.log"`)
  - Path to the spooler log file
  - Parent directory will be created automatically
  - `koco` option `--log-file` (spool command)
    - Can override `KODO_SPOOLER_LOG_FILE` in spool command

- `SPOOLER_LOG_FILE_LEVEL` (default: `"DEBUG"`)
  - Log level for the spooler log file
  - Available levels: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
  - `koco` option `--log-file-level` (spool command)
    - Can override `KODO_SPOOLER_LOG_FILE_LEVEL` in spool command

- `SPOOLER_STD_LEVEL` (default: `"INFO"`)
  - Log level for spooler console output
  - Available levels: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
  - `koco` option `--level` (spool command)
    - Can override `KODO_SPOOLER_STD_LEVEL` in spool command

- `SPOOLER_INTERVAL` (default: `0.25`)
  - Polling interval for the spooler in seconds
  - Must be greater than 0
  - `koco` option `--interval` (spool command)
    - Can override `KODO_SPOOLER_INTERVAL` in spool command

- `SPOOLER_BATCH_SIZE` (default: `10`)
  - Number of items to process in each batch
  - Must be at least 1
  - `koco` option `--batch-size` (spool command)
    - Can override `KODO_SPOOLER_BATCH_SIZE` in spool command

- `SPOOLER_BATCH_TIMEOUT` (default: `0.1`)
  - Timeout for batch retrieval in seconds
  - Must be greater than 0
  - `koco` option `--timeout` (spool command)
    - Can override `KODO_SPOOLER_BATCH_TIMEOUT` in spool command

### Ray Settings

- `RAY_SERVER` (default: `"localhost:6379"`)
  - Ray server URL
  - `koco` option `--ray-server` (spool command)
    - Can override `KODO_RAY_SERVER` in spool command

- `RAY_DASHBOARD` (default: `"http://localhost:8265"`)
  - Ray dashboard URL

### Application Server Settings

- `APP_SERVER` (default: `"http://localhost:3370"`)
  - Application server URL
  - `koco` option `--address` (serve command)
    - Can override `KODO_APP_SERVER` in serve command

- `APP_LOG_FILE` (default: `"./data/app.log"`)
  - Path to the application log file
  - `koco` option `--log-file` (serve command)
    - Can override `KODO_APP_LOG_FILE` in serve command

- `APP_LOG_FILE_LEVEL` (default: `"DEBUG"`)
  - Log level for the application log file
  - Available levels: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
  - `koco` option `--log-file-level` (serve command)
    - Can override `KODO_APP_LOG_FILE_LEVEL` in serve command

- `APP_STD_LEVEL` (default: `"INFO"`)
  - Log level for application console output
  - Available levels: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
  - `koco` option `--level` (serve command)
    - Can override `KODO_APP_STD_LEVEL` in serve command

- `APP_RELOAD` (default: `False`)
  - Enable auto-reload on file changes
  - `koco` option `--reload` (serve command)
  - Can override `KODO_APP_RELOAD` in serve command

- `UVICORN_LEVEL` (default: `"WARNING"`)
  - Log level for Uvicorn server
  - Available levels: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
  - `koco` option `--uvicorn-level` (serve command)
    - Can override `KODO_UVICORN_LEVEL` in serve command

### Security Settings

- `SECRET_KEY`
  - Secret key for the application
  - Should be changed in production

- `CORS_ORIGINS` (default: `["*"]`)
  - List of allowed CORS origins
  - Can be set as comma-separated string

### Database Settings

- `ADMIN_DATABASE` (default: `"sqlite+aiosqlite:///./data/admin.db"`)
  - Database URL for admin database

- `ADMIN_EMAIL`
  - Admin user email
  - Should be changed in production

- `ADMIN_PASSWORD` (default: `"admin"`)
  - Admin user password
  - Should be changed in production

### Ray Serve Settings

- `RAY_SERVE_ADDRESS` (default: `"http://localhost:8005"`)
  - Ray Serve HTTP endpoint address
  - Used for proxying requests to deployed agent services

- `SERVE_EXECUTABLE` (default: `"serve"`)
  - Path to the Ray Serve executable

- `BOOT_HEALTH_TIMEOUT` (default: `1800`)
  - Boot process timeout in seconds (30 minutes)

### Sumi Protocol Settings

- `SUMI_ADDRESS` (default: `""`)
  - Public URL for the Sumi API
  - Falls back to `APP_SERVER` if empty
  - Used as base URL for agent `apiBaseUrl` in registry

### Masumi Payment Integration

- `MASUMI` (default: `""`)
  - Masumi network configuration (space-separated: `"Name URL Token [pay_by_time] [submit_result_by_time] [poll_interval]"`)
  - Supports multiple networks via `MASUMI0` through `MASUMI9` (or `KODO_MASUMI0` through `KODO_MASUMI9`)
  - Example: `KODO_MASUMI="Preprod https://payment.masumi.network/api/v1 mytoken"`
  - Example multi-network:
    ```
    KODO_MASUMI0="Preprod https://payment.masumi.network/api/v1 preprod-token"
    KODO_MASUMI1="Mainnet https://mainnet-payment.example.com/api/v1 mainnet-token 1200 3600 1.0"
    ```
  - Optional fields: `pay_by_time` (default: 1200s), `submit_result_by_time` (default: 3600s), `poll_interval` (default: 1.0s)

### File Upload Settings

- `UPLOAD_DIR` (default: `"./data/uploads"`)
  - Directory for temporary file uploads

- `CHUNK_SIZE` (default: `5242880` / 5MB)
  - Chunk size for file upload operations in bytes

- `SAVE_CHUNK_SIZE` (default: `1048576` / 1MB)
  - Chunk size for saving files to disk in bytes

### Audit Logging Settings

- `AUDIT_LOG_FILE` (default: `"./data/audit.log"`)
  - Path to the audit log file for boot/deployment events

- `AUDIT_LOG_MAX_BYTES` (default: `10485760` / 10MB)
  - Maximum size per audit log file before rotation

- `AUDIT_LOG_BACKUP_COUNT` (default: `5`)
  - Number of rotated audit log backup files to keep

### Log Rotation Settings

- `SPOOLER_LOG_MAX_BYTES` (default: `10485760` / 10MB)
  - Maximum size per spooler log file before rotation

- `SPOOLER_LOG_BACKUP_COUNT` (default: `5`)
  - Number of rotated spooler log backup files to keep

- `APP_LOG_MAX_BYTES` (default: `10485760` / 10MB)
  - Maximum size per app log file before rotation

- `APP_LOG_BACKUP_COUNT` (default: `5`)
  - Number of rotated app log backup files to keep

### SSL Settings

- `SSL_KEYFILE` (default: `None`)
  - Path to SSL key file

- `SSL_CERTFILE` (default: `None`)
  - Path to SSL certificate file

- `SSL_KEYFILE_PASSWORD` (default: `None`)
  - Password for the SSL key file

- `SSL_CA_CERTS` (default: `None`)
  - Path to CA certificates file

### Lock and Session Settings

- `LOCK_EXPIRES` (default: `10800` / 3 hours)
  - How long a HITL lock remains valid in seconds

- `ANONYMOUS` (default: `"_ANONYMOUS_"`)
  - Identifier used for anonymous (non-authenticated) users

### Other Settings

- `WAIT_FOR_JOB` (default: `600`)
  - Maximum wait time for job database to appear in seconds

- `PROXY_TIMEOUT` (default: `30`)
  - HTTP proxy request timeout in seconds

- `APP_WORKERS` (default: `1`)
  - Number of Uvicorn worker processes

- `YAML_BASE` (default: `"./data/config/config.yaml"`)
  - Path to the base YAML configuration file

- `REGISTER_FLOW` (default: `[]`)
  - List of flow URLs to auto-register on startup
